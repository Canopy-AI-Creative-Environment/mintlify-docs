## API Keys

Generate API keys from **Workspace Settings > API** in your Canopy AI dashboard at [app.trycanopy.space](https://app.trycanopy.space). Each key is scoped to a single workspace.

<Note>Keep your API keys secret. Do not commit them to version control or expose them in client-side code.</Note>

## Authentication

Include your API key as a Bearer token in the `Authorization` header of every request:

```bash
curl https://developer.trycanopy.space/api/v1/techniques \
  -H "Authorization: Bearer YOUR_CANOPY_API_KEY"
```

## Environment Variables

Store your key in an environment variable for convenience:

```bash
export CANOPY_API_KEY="your-api-key-here"
```

Then reference it in your code or scripts:

```bash
curl https://developer.trycanopy.space/api/v1/techniques \
  -H "Authorization: Bearer $CANOPY_API_KEY"
```

## Key Management

| Action | How |
|---|---|
| **Create** | Workspace Settings > API > **Generate Key** |
| **Revoke** | Click the trash icon next to the key |
| **Rotate** | Revoke the old key, generate a new one |

<Tip>Create separate API keys for development and production environments so you can rotate them independently.</Tip>
