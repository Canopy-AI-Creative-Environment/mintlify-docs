## Overview

The Batch Node lets you process multiple inputs through the same downstream workflow in parallel. Feed it a list of prompts, images, or documents and it fans out each item to your connected nodes, then collects the results.

<Tip>Batch Nodes are ideal for producing variations -- feed 10 different prompts into an Image Node to generate 10 images in one run.</Tip>

## How It Works

Connect a data source (text list, multiple images, or a Document Node with multiple pages) to the Batch Node's input. The Batch Node sends each item individually to all connected downstream nodes, runs them in parallel, and aggregates the outputs.

## Configuration

| Setting | Description |
|---|---|
| **Concurrency** | Number of items processed simultaneously (higher = faster, more credits) |
| **Input mode** | List (one per line), JSON array, or connected multi-output node |
| **Error handling** | Skip failed items or halt the entire batch |

<Note>Batch processing consumes credits for each individual generation. A batch of 10 image generations costs the same as running 10 separate Image Nodes.</Note>
