## Overview

The Router Node takes a single input and forwards it to multiple downstream nodes. Use it to fan out data across parallel processing branches without duplicating upstream work.

## Use Cases

<CardGroup cols={2}>
  <Card title="Multi-Format Output">Route one prompt to an Image Node, a Video Node, and a Text Node simultaneously.</Card>
  <Card title="A/B Testing">Send the same input to two Image Nodes with different models or settings to compare results.</Card>
  <Card title="Multi-Language">Route text to multiple Text Nodes, each configured for a different target language.</Card>
  <Card title="Redundancy">Process the same input through different pipelines and pick the best output.</Card>
</CardGroup>

## How It Works

Connect an upstream node's output to the Router Node's input port. Then connect the Router's output port to as many downstream nodes as you need. When the workflow runs, the Router passes the input data to every connected downstream node.

<Tip>Combine a Router Node with an Export Node at each branch to save all variations in one run.</Tip>
