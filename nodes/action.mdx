Action Nodes bring traditional editing tools into your FLORA canvas as first-class workflow steps. Instead of exporting media to a separate app and reimporting the result, you drop an Action Node into your workflow, connect it like any other node, and get deterministic, repeatable edits — right alongside your generation nodes, Techniques, and Batch Nodes.

## What Actions Cover

Actions handle the kinds of operations you'd normally reach for a dedicated editing app to perform:

- **Adjusting color and tone** on an image or video
- **Splitting or stitching** video clips together
- **Extracting frames** from a video as still images
- **Changing aspect ratios** with crop or pad options
- **Manipulating text** with find-and-replace, splitting, or concatenation
- **Splitting or merging audio tracks** from video files

## Adding an Action Node

<Steps>
  <Step title="Open the Add Node menu">
    Press the **+** button or use the canvas shortcut to open the node menu. Under the **Utilities** section, click **Action**.
  </Step>
  <Step title="Choose an action">
    A picker panel opens showing all available actions. Browse or search by name. Each action displays its input and output modalities (image, video, or text) so you know exactly what to connect.
  </Step>
  <Step title="Connect your inputs">
    Drag a noodle from any upstream node to the Action Node's input handle. The action processes whatever media is connected.
  </Step>
  <Step title="Configure parameters">
    Each action exposes its own set of controls — sliders, dropdowns, color pickers, and toggles. Adjust them to dial in the result you want.
  </Step>
  <Step title="Run">
    Click **Run** on the Action Node. Results appear in one or more output slots depending on the action.
  </Step>
</Steps>

## Using Actions in Workflows

### Chaining with Generation Nodes

Actions slot naturally before or after AI generation for consistent, branded results:

```
[Text Node] → [Image Node: Generate] → [Color Grade Image] → Final Output
              "product on white bg"     warmth=0.2, contrast=1.3
```

### Combining with Techniques

Reformat or prepare media before feeding it into a multi-step Technique:

```
[Image Upload] → [Change Image AR] → [Technique: Product Lifestyle] → Final Output
                  9:16 portrait
```

### Actions Inside Techniques

You can include Action Nodes directly inside a Technique's workflow graph. The action executes via the sandbox as part of the Technique run, and its outputs flow through to downstream nodes automatically. This lets you package deterministic post-processing — color grading, frame extraction, audio splitting — into a reusable pipeline without asking users to configure those steps separately.

### Batch Processing

Connect a Batch Node upstream of an Action Node to apply the same operation across many items at once:

```
[Image 1] ─┐
[Image 2] ─┼─→ [Batch Node] ─→ [Color Grade Image] → 4 graded outputs
[Image 3] ─┤     (4 items)      warmth=0.15
[Image 4] ─┘
```

Every item in the batch is processed with identical settings — perfect for maintaining visual consistency across a set of assets.

### Audio Workflows

The audio actions enable creative audio-visual pipelines:

```
[Video Upload] → [Split Audio from Video] → Audio track + muted video
```

```
[Video] ─→ [Merge Audio into Video] → Video with new soundtrack
[Audio] ─┘
```

## Tips

<Tip>
  Test your action on a single input first to dial in the parameters, then connect a Batch Node to scale across your full collection.
</Tip>

- **Chain actions freely** — they're lightweight and fast, so stacking multiple actions adds minimal processing time
- **Use Duplicate Image / Video** to branch a single input into parallel workflow lanes with different downstream processing
- **Dynamic outputs** (like Split Video or Extract Frames) produce multiple result nodes — each can feed its own downstream workflow
- **Audio actions stream-copy** when source codecs are MP4-native (e.g., h264 + AAC), making them near-instant for common video formats
